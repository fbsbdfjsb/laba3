Алгоритм — это точная последовательность команд, которая от начальных данных приводит к нужному результату за конечное число шагов.

**Блок-схемы**.

В русской Википедии статья "Блок-схема" описывает стандартные обозначения:

овалы для начала/конца, прямоугольники для действий/процессов, ромбы для условий/решений, параллелограммы для ввода/вывода данных, стрелки для потока выполнения.


С помощью интеллектуального помощника perplexity, узнала когда «блок-схемы алгоритмов» стали называть «схемы алгоритмов» и почему:

"Блок-схемы алгоритмов" стали официально называть "схемами алгоритмов" в СССР с 1980 года по ГОСТ 19.002-80 ЕСПД, для унификации с другими схемами (электрическими, логическими) в единой системе; термин "блок-схема" остался в общеупотреблении.

**Отечественный стандарт по рисованию схемы алгоритма**

Стандарт — ГОСТ 19.701-90 "ЕСПД. Схемы алгоритмов, программ, данных и систем. Условные обозначения и правила выполнения". В конце — примеры: прямоугольники с закругленными углами для команд, ромбы для ветвлений, стрелки слева/сверху внутрь, справа/снизу наружу.

**Workflow Diagram и связь с Flowchar**

Workflow Diagram (WFD) — диаграмма рабочих процессов, визуализирующая последовательность задач, ролей и решений в бизнесе (с "дорожками" для исполнителей).
WFD — разновидность Flowchart, адаптированная для workflow (потоков работ), с похожими символами (ромбы, прямоугольники), но фокусом на бизнес-процессах и участниках, а не чистых алгоритмах.

**BASE64 + API**

Base64 — это способ закодировать двоичные данные (байты) в обычный текст, чтобы их можно было безопасно передавать там, где разрешены только печатные символы (например, в URL, e‑mail, JSON).

Основная идея Base64
- Данные берутся по 3 байта (24 бита) и делятся на группы по 6 бит, каждые 6 бит превращаются в один символ из специального набора — «алфавита» Base64.  
- Число 64 в названии означает, что алфавит содержит 64 разных символа, то есть это система счисления с основанием 64 (radix‑64).
Алфавит Base64 и почему 64 символа
- Стандартный Base64-алфавит включает:  
  - 26 заглавных букв: A–Z  
  - 26 строчных букв: a–z  
  - 10 цифр: 0–9  
  - 2 спецсимвола: + и /  
  Итого 26 + 26 + 10 + 2 = 64 символа.\
- 64 — это \(2^6\), поэтому 1 символ Base64 кодирует ровно 6 бит, а каждые 4 символа Base64 представляют 24 бита, то есть ровно 3 исходных байта — удобно для передачи байтовыми потоками.

С помощью интеллектуального помощника выяснила, чем занимается специалист по выбранной профессии(Full-Stack разработчик):

<br>
<br>

Full-Stack разработчик — это универсальный IT-специалист, который создает и поддерживает полные веб-приложения, охватывая как клиентскую (frontend), так и серверную (backend) стороны, а также интеграцию с базами данных.

## 1. Краткое определение профессии
Full-Stack разработчик отвечает за весь стек технологий веб-приложения — от интерфейса пользователя до серверной логики и хранения данных. Это специалист, способный самостоятельно реализовать проект "от идеи до деплоя", часто работая в малых командах или стартапах.

## 2. Список ключевых обязанностей (10-15 пунктов)
1. Проектирование архитектуры приложения: выбор стека технологий (React + Node.js), определение API endpoints.[1]
2. Разработка frontend: создание responsive UI с HTML/CSS/JS, фреймворками React/Vue/Angular.
3. Реализация backend: написание серверного кода на Node.js/Python (Django/Flask)/Java (Spring), обработка запросов.
4. Интеграция баз данных: проектирование схем SQL (PostgreSQL/MySQL) или NoSQL (MongoDB), написание запросов.
5. Создание REST/GraphQL API: endpoints для CRUD-операций, аутентификация JWT/OAuth.
6. Тестирование: unit-тесты (Jest), интеграционные (Postman), e2e (Cypress).
7. Деплой и CI/CD: настройка Docker, Kubernetes, GitHub Actions, облака (AWS/Heroku).
8. Оптимизация производительности: профилирование, кэширование (Redis), lazy loading.
9. Работа с версионным контролем: Git branching, merge requests, code review.
10. Мониторинг и логирование: Sentry, Prometheus, обработка ошибок в production.
11. Интеграция third-party сервисов: платежи (Stripe), уведомления (Firebase), аналитика (Google Analytics).
12. Рефакторинг legacy-кода: миграция монолита на микросервисы.
13. Документирование: Swagger для API, README для репозитория.
14. Участие в планировании: оценка задач в Jira/Trello, ретроспективы.
15. Поддержка production: hotfixes, scaling под нагрузку.

## 3. Типичные проекты и роли в команде
Проекты: e-commerce платформы, SaaS-сервисы, dashboards, социальные сети. Роли: lead developer в стартапе (80% задач сам), mid-level в корпорации (frontend/backend по ротации), ментор junior'ов.

## 4. Разделение задач
- **Frontend**: UI/UX (70% времени на верстку/логику), state management (Redux).
- **Backend**: бизнес-логика (аутентификация, валидация), security (защита от SQLi/XSS).
- **Full-stack**: DevOps (деплой), full-cycle фичи (от тикета до релиза).[1]
